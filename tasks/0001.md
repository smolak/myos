## Steps

### Part A: Delete stale files and artifacts

1. Delete `artifacts/` directory (stale build outputs, ~36MB)
2. Delete `build/` directory (stale build outputs)
3. Delete `dist/` directory (stale Vite output)
4. Delete `Archiwum.zip` (leftover archive, not part of the project)
5. Delete `the.gitignore` (duplicate of `.gitignore`)
6. Delete `src/mainview/App.tsx` (demo counter component)

### Part B: Create new directory structure

1. Create `src/core/bun/` with `.gitkeep`
2. Create `src/core/types/` with `.gitkeep`
3. Create `src/core/ui/` with `.gitkeep`
4. Create `src/shell/bun/`
5. Create `src/shell/view/`
6. Create `src/shell/shared/` with `.gitkeep`
7. Create `src/features/` with `.gitkeep`
8. Create `src/scripts/` with `.gitkeep`

### Part C: Move and rewrite source files

#### C1. Shell — Bun entry (`src/shell/bun/main.ts`)

Rewrite `src/bun/index.ts` → `src/shell/bun/main.ts`.

The new main process should:

- Create a `BrowserWindow` with a title reflecting the app name (not "React + Tailwind + Vite")
- Set up a `Tray` with show/hide/quit options
- Set `runtime.exitOnLastWindowClosed: false` (so the app stays alive when window is hidden)
- Wire window close to hide instead of quit
- Load from `views://dashboard/index.html` (not `mainview`)
- Keep the Vite HMR dev-server detection logic (it's useful for development)
- Log startup confirmation

After this step, delete `src/bun/index.ts` and the `src/bun/` directory.

#### C2. Shell — View entry (`src/shell/view/`)

Move and rewrite the view layer:

- `src/shell/view/index.html` — updated HTML with new title, same structure
- `src/shell/view/main.tsx` — React root mount (from `src/mainview/main.tsx`)
- `src/shell/view/App.tsx` — empty shell component. Should render:
  - A placeholder header with the app name
  - A message: "Dashboard — no features installed yet"
  - Styled with Tailwind (clean, minimal, dark-mode-ready)
- `src/shell/view/index.css` — moved from `src/mainview/index.css` (just the Tailwind import)

After this step, delete `src/mainview/` directory entirely.

### Part D: Update configuration files

#### D1. `electrobun.config.ts`

Update:

- `app.name`: choose a working project name (e.g., `"nexus"`, `"dash"`, or any placeholder — can be changed later)
- `app.identifier`: update to match (e.g., `"app.nexus.local"`)
- `app.version`: keep `"0.0.1"`
- `build.copy`: update paths to match new `dist/` output location mapping to `views/dashboard/`
- Keep `bundleCEF: false` on all platforms

#### D2. `vite.config.ts`

Update:

- `root`: change from `"src/mainview"` to `"src/shell/view"`
- `build.outDir`: adjust relative path to `../../dist` (verify it still resolves to project-root `dist/`)

#### D3. `tailwind.config.js`

Update:

- `content` array: change from `"./src/mainview/**/*.{html,js,ts,jsx,tsx}"` to include all source paths:
  - `"./src/shell/view/**/*.{html,js,ts,jsx,tsx}"`
  - `"./src/features/**/view/**/*.{html,js,ts,jsx,tsx}"`
  - `"./src/core/ui/**/*.{html,js,ts,jsx,tsx}"`

#### D4. `tsconfig.json`

Update:

- `include` should cover `"src"` (already does, no change needed if all code stays under `src/`)
- Consider adding path aliases for cleaner imports later (not required now, but note for future):
  - `"@core/*"` → `"src/core/*"`
  - `"@shell/*"` → `"src/shell/*"`
  - `"@features/*"` → `"src/features/*"`

#### D5. `package.json`

Update:

- `name`: change from `"electrobun-react-tailwind-vite"` to something meaningful (e.g., `"nexus"` or the chosen app name)
- `description`: change from `"Electrobun app with React, Tailwind CSS, and Vite HMR"` to something like `"Local-first productivity dashboard built with Electrobun"`
- `scripts`: update `hmr` script if Vite root path changed
- Do NOT add new dependencies yet — that's a separate task

#### D6. `.gitignore`

Append these entries if not already present:

```gitignore

# SQLite databases anywhere else (safety net)
*.db
*.db-wal
*.db-shm

# User data scripts (exported from DB for backup — not source code)
src/scripts/local/

# Backups
backups/
```

#### D7. `README.md`

Rewrite entirely. Should contain:

- Project name and one-line description
- Link to `specs/ARCHITECTURE.md` for full design
- Getting started: `bun install`, `bun run dev`, `bun run dev:hmr`
- Link to `specs/SKILLS.md` for AI skills guide
- Brief note: "See `tasks/` for the development roadmap"

### Part E: Verify everything works

1. Run `bun install` to ensure dependencies resolve
2. Run `bun run dev` (or `bun run start`) to confirm the app launches
3. Verify the window shows the empty dashboard shell (not the demo counter)
4. Verify the tray icon appears and show/hide/quit work
5. Verify window close hides the window instead of quitting
6. Verify `bun run dev:hmr` works with Vite HMR
